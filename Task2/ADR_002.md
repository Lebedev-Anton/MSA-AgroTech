# Решение

Диаграмма контейнеров: C2.puml

Было принято решение использовать следующие технологии:

Kafka как единый Event Store: Использование Kafka для всех событийных потоков обеспечивает гарантированную доставку, возможность реплея для восстановления состояния агентов и высокую пропускную способность. У компании есть экспертиза

ClickHouse для аналитики: Стандартное решение OLAP-нагрузок, так же имеется опыт использования судя по архитектуре компании.

Python: Единообразие стека разработки упрощает поддержку и позволяет быстро адаптировать сервисы. Python является естественным выбором для нейросетевой аналитики.

В совокупности эти решения обеспечивают автономность фермы, низкую задержку реакции и высокую скорость разработки.

# Альтернативное решение

Альтернативное решение предполагает полностью централизованную архитектуру, где видеоаналитика, управление IoT и оповещения выносятся в облако. На ферме остаётся только тонкий шлюз на Python для маршрутизации видеопотоков и телеметрии.

Также альтернативное решение включает использование MinIO/S3 для хранения видеозаписей в облаке и Notification Gateway для отправки push-уведомлений.
